# Nolan Cloud Platform (NCP) Plan

2주간의 AWS 스타일 클라우드 플랫폼 구축 계획입니다.

## 1. 프로젝트 목표
- 핵심 클라우드 서비스(Compute, Storage, Network)의 추상화 레이어 구축
- 사용자 친화적인 CLI 또는 Dashboard 제공
- 확장 가능한 마이크로서비스 아키텍처 지향

## 2. 핵심 기능 (MVP)
- **Compute (N-EC2)**: 컨테이너(Docker) 기반의 가상 인스턴스 생명주기 관리 (생성, 중지, 삭제)
- **Storage (N-S3)**: 파일 업로드/다운로드 및 버킷 관리
- **Network**: 가상 네트워크 인터페이스 및 포트 포워딩 설정
- **Identity (N-IAM)**: 기본적인 사용자 인증 및 토큰 기반 권한 관리

## 3. 작업 순서 (2주 스케줄)

### 1주차: Infrastructure & Core API
- **Day 1**: 요구사항 분석 및 상세 아키텍처 설계 (완료)
- **Day 2-3**: Core API 서버 구축 (Go 또는 Node.js) 및 Database 스키마 정의
- **Day 4-5**: Compute 엔진 구현 (Docker SDK 연동)
- **Day 6-7**: Storage 서비스 구현 (Local 파일 시스템 기반 S3 호환 API)

### 2주차: Integration & UI/CLI
- **Day 8-9**: Network 및 보안 그룹 설정 기능 추가
- **Day 10-11**: CLI 툴 (`ncp`) 개발 - 리소스 제어용
- **Day 12-13**: 통합 테스트 및 버그 수정
- **Day 14**: 문서화 및 최종 데모 준비

## 4. 아키텍처 구성 (초안)
- **Frontend**: React 기반 Dashboard (선택 사항)
- **API Gateway**: 모든 요청의 진입점 및 인증 처리
- **Controller**: 각 서비스(Compute, Storage)의 로직을 담당하는 Worker
- **Provider**: 실제 리소스(Docker Engine, File System)와 상호작용하는 레이어
- **Database**: PostgreSQL (상태 관리용)
